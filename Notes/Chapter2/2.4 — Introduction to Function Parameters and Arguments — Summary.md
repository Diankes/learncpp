# 2.4 ‚Äî Introduction to Function Parameters and Arguments ‚Äî Summary

---

# The Problem

Functions can return values, but sometimes they also need **input** to work with.

Example problem:

- `printDouble()` couldn‚Äôt access `num`
    
- Functions **cannot see variables** from other functions
    
- Each function has its own scope
    

So we need a way to **pass data into functions**.

---

# Function Parameters vs Arguments

### üîπ Parameter

A variable defined in the **function header**.

```cpp
void printValue(int x)
```

`x` is a **parameter**.

---

### üîπ Argument

The actual value passed to the function when calling it.

```cpp
printValue(6);
```

`6` is the **argument**.

---

# How It Works (Pass by Value)

When a function is called:

1. Parameters are created
    
2. Arguments are copied into those parameters
    
3. The function runs using those copies
    

Example:

```cpp
void printValues(int x, int y)
{
    std::cout << x << '\n';
    std::cout << y << '\n';
}

printValues(6, 7);
```

Inside the function:

```
x = 6
y = 7
```

This is called **pass by value**.

---

# Fixing the Original Problem

Old broken version:

```cpp
void printDouble()
{
    std::cout << num << " doubled is: " << num * 2;
}
```

Correct version:

```cpp
void printDouble(int value)
{
    std::cout << value << " doubled is: " << value * 2;
}

printDouble(num);
```

Now the function receives the number as a parameter.

---

# Using Return Values as Arguments

Instead of:

```cpp
int num { getValueFromUser() };
printDouble(num);
```

We can write:

```cpp
printDouble(getValueFromUser());
```

The return value becomes the argument directly.

---

# Parameters + Return Values Together

Functions can:

1. Take input (parameters)
    
2. Process it
    
3. Return a result
    

Example:

```cpp
int add(int x, int y)
{
    return x + y;
}

std::cout << add(4, 5);
```

Flow:

```
add(4, 5)
‚Üí x = 4, y = 5
‚Üí returns 9
‚Üí printed
```

---

# Arguments Can Be Expressions

Arguments don‚Äôt need to be simple numbers.

These are valid:

```cpp
add(1 + 2, 3 * 4);
add(a, a);
add(1, multiply(2, 3));
add(1, add(2, 3));
```

The expressions are evaluated **before** being passed.

Example:

```
add(1, multiply(2, 3))
‚Üí multiply(2, 3) = 6
‚Üí add(1, 6) = 7
```

---

# Argument Count Must Match

Generally:

- Number of arguments must match number of parameters
    
- Otherwise ‚Üí compile error
    

---

# Unreferenced Parameters

Sometimes a function has a parameter it doesn‚Äôt use:

```cpp
void doSomething(int count)
{
}
```

This causes a warning.

Solution:

Use an **unnamed parameter**:

```cpp
void doSomething(int)
{
}
```

Or:

```cpp
void doSomething(int /*count*/)
{
}
```

Best practice:

> If a parameter isn‚Äôt used, don‚Äôt name it.

# Common Errors (From Quiz)

### ‚ùå Returning a value from a void function

```cpp
void multiply(int x, int y)
{
    return x * y; // error
}
```

Fix: change return type to `int`.

---

### ‚ùå Wrong number of arguments

```cpp
multiply(4); // function expects 2 arguments
```

---

### ‚ùå Missing return statement

```cpp
int multiply(int x, int y)
{
    int product { x * y };
    // missing return
}
```

---

# Key Takeaways

- **Parameters** = variables in function header
    
- **Arguments** = values passed to function
    
- C++ uses **pass by value** (copies are made)
    
- Argument count must match parameter count
    
- Arguments can be expressions or function calls
    
- Functions can both **take input and return output**
    
- Unused parameters should be unnamed
    

---

# Big Idea

Parameters and return values are what make functions:

- Reusable
    
- Flexible
    
- Modular
    
- Independent
    

Without them, functions wouldn‚Äôt be able to work with dynamic data.
