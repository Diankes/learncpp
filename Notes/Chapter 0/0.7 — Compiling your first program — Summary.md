## 0.7 — Compiling your first program — Summary

This lesson explains how to create, compile, and run a simple C++ program using an IDE or the command line.

### Projects

- A **project** is a container for all files and settings needed to produce an executable.  
    
- All `.cpp` files in a project are compiled and linked together.  
    
- Each project corresponds to one program.  
    
- Best practice: create a new project for each new program.  
    
- Project files are IDE-specific.  
    

### Console projects

- All programs in this tutorial are **console applications**.  
    
- Console programs:
    - Have no GUI  
        
    - Print text to the console  
        
    - Read keyboard input  
        
    - Produce standalone executables  
        
- They are simpler and ideal for learning.  
    

### Workspaces / Solutions

- A workspace (or solution) can contain multiple related projects.  
    
- While learning, it is recommended to create a new workspace/solution per program.  
    

### Creating and running Hello World

Across IDEs (Visual Studio, Code::Blocks, VS Code):

- Create a new console project.  
    
- Disable **precompiled headers** (not useful for small projects).  
    
- Replace the auto-generated code with:  
    

```cpp
#include <iostream>

int main()
{
    std::cout << "Hello, world!";
    return 0;
}
```

- Build/compile the project.  
    
- Run the executable.  
    

Command line (g++):

- Save file as `HelloWorld.cpp`  
    
- Compile: `g++ -o HelloWorld HelloWorld.cpp`  
    
- Run: `HelloWorld` (or `./HelloWorld`)  
    

### Precompiled headers

- Speed up compilation in large projects.  
    
- Not useful for small tutorial programs.  
    
- Should be disabled initially.  
    

### If compilation fails

Recommended troubleshooting steps:

1. Check the error message and line numbers.
2. Review common problems (lesson 0.8).
3. Read lesson comments.
4. Search the error message online.

### If console closes immediately

- Some IDEs close the console after execution.  
    
- To pause at program end, add:  
    

```cpp
#include <limits>
```

And before `return` in `main()`:

```cpp
std::cin.clear();
std::cin.ignore(std::numeric_limits<std::streamsize>::max(), '\n');
std::cin.get();
```

- Avoid `system("pause")` (not portable).  
    

### Antivirus issues

- Antivirus software may block execution.  
    
- Consider excluding the project directory from scans.  
    

### IDE build-related terms

- **Build**: Compile modified files and link.  
    
- **Clean**: Remove cached object files and executables.  
    
- **Rebuild**: Clean + Build.  
    
- **Compile**: Compile a single file (no linking).  
    
- **Run/Start**: Execute the built executable (some IDEs auto-build first).  
    

### Conclusion

This lesson teaches how to:

- Create a console project  
    
- Compile source files  
    
- Link them into an executable  
    
- Run the program  
    
- Troubleshoot common setup and execution issues
