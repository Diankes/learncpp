# 3.9 — The Call Stack

The call stack keeps track of all the active functions that have been called to reach the current point of execution.

- **CALL STACK Panel:** VS Code displays the call stack on the left side during debug mode. It includes an entry for each function called, as well as the line of code that will be returned to.
    
- **Reading the Stack:** The top entry represents the currently executing function. The line number shown indicates the next line that will execute when execution resumes.
    
- **Multi-threading:** The C/C++ extension in VS Code supports debugging multi-threaded programs, displaying all threads and their respective call stacks in this panel.
