# 2.3 — Void functions (Non-value returning functions) — Summary

---

# What Is a Void Function?

A function with return type `void`:

```cpp
void printHi()
{
    std::cout << "Hi\n";
}
```

Means:

> This function does NOT return a value to the caller.

It performs an action (behavior), but sends nothing back.

Such functions are called:

- **Non-value returning functions**
    
- **Void functions**
    

---

# How Void Functions Work

When called:

```cpp
printHi();
```

Flow:

1. Execution jumps to `printHi`
    
2. Code runs
    
3. Control automatically returns to caller
    

No return value is produced.

---

# Return Statements in Void Functions

A void function:

- Does **not require** a `return` statement.
    
- Automatically returns at the end.
    

You _can_ write:

```cpp
return;
```

But:

> Putting `return;` at the end is redundant.

### Best Practice

Do not put a `return;` at the end of a void function.

---

# Void Functions and Expressions

Some contexts **require a value**, others don’t.

---

## ✔ Valid Use

Calling function by itself:

```cpp
printHi();
```

No value required → OK.

---

## ❌ Invalid Use

```cpp
std::cout << printHi();  // compile error
```

Why?

- `std::cout <<` requires a value
    
- `printHi()` returns nothing
    
- Compiler doesn’t know what to print
    

So this fails to compile.

---

# Important Rule

You can:

- Call a value-returning function and ignore the result
    
- Call a void function as a standalone statement
    

You cannot:

- Use a void function where a value is required
    

---

# Example Comparison

```cpp
void returnNothing() {}
int returnFive() { return 5; }
```

### Valid:

```cpp
returnNothing();      // ok
returnFive();         // ok (value ignored)
std::cout << returnFive();  // ok
```

### Invalid:

```cpp
std::cout << returnNothing(); // compile error
```

---

# Returning a Value from a Void Function

This is illegal:

```cpp
void printHi()
{
    return 5;  // compile error
}
```

If return type is `void`, you cannot return a value.

---

# Quiz Review

---

```cpp
printA();

```

Output:

```
A

```

---

## 1b

```cpp
std::cout << printA();
```

Compile error.

Because `printA()` returns `void`.

---

# Key Takeaways

- `void` means no return value
    
- Void functions are used for behavior, not data
    
- They automatically return at the end
    
- They cannot be used where a value is required
    
- They cannot return a value
    

---

# Big Picture

There are two types of functions:

1. **Value-returning functions** → produce data
    
2. **Void functions** → perform actions
    

Understanding when to use each is fundamental to writing clean, modular C++ programs.
