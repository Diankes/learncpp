# 3.7 — Running and Breakpoints

These tools help you quickly navigate to the parts of your code you want to examine.

- **Breakpoints (`F9`):** A special marker that tells the debugger to halt execution at that specific line. You can set or remove them by clicking to the left of the line number or pressing `F9`.
    
- **Conditional and Function Breakpoints:** VS Code allows you to right-click an existing breakpoint and select "Edit Breakpoint" to add a condition that must evaluate to true. You can also break at the start of a function by right-clicking in the Breakpoints section and choosing "Add Function Breakpoint".
    
- **Continue (`F5`):** Resumes execution of the program until it terminates or hits another breakpoint.
    
- **Run to Cursor:** Right-click a statement and choose "Run to Cursor" to execute the program up to that line.
    
- **Set Next Statement (Jump to Cursor):** Right-click a statement and choose "Jump to cursor" to change the point of execution. This does not rewind your program state; variables retain their current values, which can lead to undefined behavior or crashes if used improperly.
